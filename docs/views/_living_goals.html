Goals

Purpose. This file holds the project's Goals as a loose documentation container. A Goal is a human-readable statement of aspiration ("ship first revenue by X date"). Goals group Business Objectives (BOs) for narrative clarity but do NOT carry metric fields or convergence rollups. Not a hierarchy node. Goals are NOT part of the Sprint -> BO -> TBO -> TTO hierarchy. They are a parallel documentation layer. If you need metrics, use BOs in _project_hierarchy.md. Contrast with other files: - _living_goals.md (this file) - narrative descriptions of what the project aims for - _project_hierarchy.md - the measurable Sprint -> BO -> TBO -> TTO tree (where convergence is computed) - _living_decisions.md - locked decisions
